01. 자동 레이블링 모델 학습

  1. 문장 의미 분류 데이터를 기반으로 한 단일 레이블 분류 모델을 학습하고 레이블 예측 확률 값 중 엔트로피가 가장 높을 때의 확률보다 높은 상위 레이블을 최대 3개를 추출한다.

  2. 문장 의미 분류 평가 데이터에 대해 자동 레이블링을 수행하고, 추출된 멀티 레이블에 기존 정답 레이블이 포함되는지를 확인하여 자동 레이블링 모델의 신뢰도를 정량화한다.

  3. 앞 뒤 문맥 없이 독립된 하나의 문장만 입력으로 주었을 때 '후속연구'와 같은 레이블이 서론에 나타나는 등의 문제가 존재한다.

  4. 문맥 정보를 주기 위해 논문 문장의미 데이터에 문서 구조 정보를 추가하여 분류 실험을 진행하였다.

  5. 단일 레이블 평가 데이터의 성능 차이는 크지 않다.

  6. "후속연구"가 서론에 나타날 빈도가 적은 것과 같이 문장 구조에 따라 나타나는 문장의 의도가 구분될 수 있어, 섹션별 레이블 분포를 확인하여 다중 레이블이 적절하게 할당되었는지 확인한다.

    1. 한 문장에 대한 다중 레이블의 레이블들은 독립적으로 계산된다.

    2. "서론"은 레이블링된 문장이 속한 섹션이 ["서론", "머리말", "서언", "introduction"]에 포함되는 경우이다.

    3. "결론"은 레이블링된 문장이 속한 섹션이 ["결론", "맺음말", "결과", "고찰", "conclusion"]에 포함되는 경우이다.

    4. "제안 방법"은 "서론", "결론"에 포함되지 않는 문장이 해당되며, 이 중 실제로 서론에 포함되지만 2의 조건이 만족되지 않아 제안 방법으로 분류된 문장이 포함될 수 있다.

  7. 아래 표의 각 레이블 출현 비율은 논문 전문 데이터에 대해 자동 레이블링한 결과에서 확인한 출현 빈도 비율을 나타낸다.

    1. 출현 비율은 섹션별로 나누어진 레이블 목록 중 해당 레이블이 차지하는 비율을 나타낸다.

    2. 분류 모델 기반 레이블의 경우 분포가 각 모델이 유사해보인다.

    3. 매뉴얼 데이터는 모델 기반 레이블과는 다른 분포를 보인다.

      1. 특히 '서론'에 나타나는 '문제 정의' 레이블 비율이 매우 높고, '후속연구'는 거의 나타나지 않는다.

      2. '결론'에서는 '문제 정의', '가설 설정'이 거의 나타나지 않는 분류 모델과 달리 매뉴얼 데이터에서는 눈에 띄는 비율을 갖는다.

  1. 섹션별 레이블 분포를 모델 별로 비교한 그래프는 다음과 같다.

  2. 문서 구조 정보를 추가한 모델인 bert+wSection의 분포가 수작업 정제 결과인 manual의 분포와 유사해질 것을 기대하였으나, 다른 모델과 비교했을 때 두드러진 차이를 보이지는 않는다.

[M3] Multimodal-to-Multimodal-Transformer-with-External-Memory

I. 개요

  1. Retriever와 Generator를 결합한 REALM 구조 기반의 M3 모델을 개발하고 실험으로 검증한다.

II. 진행 사항

01. Constrastive Learning 기반 Retriever 학습

  1. Retriever는 generator가 가지고 있지 않은 문맥을 제공하는 것을 목표로 하며, 일반적인 상황에서는 연관이 없는 문맥이지만 현재 태스크에서 연관이 있는 문맥이라면 검색 결과로 제공될 수 있어야 한다.

  2. 태스크에 적합한 문장을 검색하도록 하기 위해 Constrastive Learning을 사용한 검색기의 재사전학습을 고려한다.

  3. 분류 태스크에서 유용한 문장은 입력과 유사하며 같은 레이블을 갖는 샘플이라 가정한다.

  4. 가정에 따라 동일한 레이블을 갖는 샘플을 유사한 쌍, 다른 레이블을 갖는 샘플을 유사하지 않는 쌍으로 학습 데이터를 생성한다.

  5. 동일한 문자열에 대한 매칭 능력을 잃지 않기 위해 자기 자신의 변형에 대한 샘플을 추가하여 이 또한 유사한 쌍으로 정의한다(self-positive).

    1. 문자열을 어절로 나누어 순서를 랜덤하게 섞거나(셔플:45%), 일부(현재 실험에서는 10%) 어절을 삭제하거나(삭제:45%), 동일한 문자열을 사용(원문:10%)한다.

  6. 유사한 샘플을 구할 수 있도록 재학습한 retriever를 사용했을 때 분명한 성능 향상을 확인할 수 있었다.

  7. NER 태스크에서는 태깅 대상 단어가 나타나는 문장으로 검색 목표 샘플을 정의할 수 있다.

  8. 해당 정의에 따른 retriever 재학습 방법을 설계하고 학습하여 성능 향상에 도움이 되는지 확인할 예정이다.

02. BERT retriever + GPT2 모델 검증 실험

  1. BERT retriever가 외부 지식에서 검색한 결과와 입력에 대한 결과를 few-shot learning된 GPT2를 통해 생성한다.

  2. GPT2에 의해 생성된 결과를 기반으로 Retriever에 의해 검색된 샘플의 유사도를 업데이트한다.

  3. 샘플의 입장에서 나머지 학습 데이터는 외부 지식으로 볼 수 있으므로, 외부지식을 별개의 데이터로 분리하지 않고 학습 데이터 자체를 외부지식으로 사용한 실험을 추가로 진행한다 (자기 자신과 동일한 샘플은 검색 결과에서 제외한다).

  4. 생성 모델을 기반으로 하기 때문에 레이블에 포함되지 않는 단어들도 생성되는데, 이는 모두 오답으로 처리된다.

  5. 단일 레이블 분류가 아닐 경우 생성 방식으로 결과를 예측해야 하고, 이 경우 retriever의 검색 결과 중 정답을 정의하는 방식이 별도로 필요하다.

  6. 전체 학습 데이터를 사용한 실험을 우선으로 진행한다.

 

03. 생성 문장 기반 Retriever 업데이트 방안 탐색 실험: KLUE-NER

  1. 태스크가 단일 레이블 분류가 아닌 경우, 정답 레이블과 모델의 예측 레이블이 완전히 동일할 때 retriever 검색 결과가 적절하다고 판단하는 방법을 사용할 수 없다.

  2. 다중 레이블 분류, 문장 생성일 때는 모델 예측이 정답과 일치하는 경우가 거의 없고 데이터 내에서도 동일한 레이블을 갖는 샘플이 극히 드물다.

  3. 레이블 분류와 문장 생성 태스크를 모두 생성의 관점에서 보면 분류와 생성 태스크 모두에 동일한 방법을 적용할 수 있다.

  4. 모델 예측과 정답 간 차이를 구하기 위해 torchtext 라이브러리에서 제공하는 BLEU-Score를 사용하여 retriever token 단위로 비교한다.

  5. BLEU-Score는 0~1 사이의 값으로 구해지며, 임의로 threshold를 0.3으로 정의하여 threshold 이상의 값을 갖는 모델의 예측은 정답으로 간주하였다.

  6. 출력 유형이 문장 내 레이블, 레이블 문자열로 지정된 경우 입력에 대한 결과가 아닌 retriever에 의해 주어진 샘플 중 가장 마지막에 위치한 샘플의 레이블 값을 생성한다.

  7. Retriever는 정답을 생성하는데 도움이 되는 샘플을 검색하면 되므로, 검색된 샘플의 쌍인 레이블이 정답에 포함되기만 하면 적절한 검색 결과라 정의하도록 설계하고, 해당 실험을 진행할 예정이다.

04. Downstream Task 정보 전달을 위한 Generator 모델 pre-tuning

  1. Retriever가 부족한 문맥을 채워준다고 하더라도 PLM이 downstream task에 대한 정보를 전혀 가지고 있지 못하면 해당 태스크를 수행하기 어렵다.

  2. Downstream Task의 입출력 형식과 같은 정보를 PLM에 전달하기 위해 아주 적은 데이터로 PLM을 fine-tuning하였다.

  3. 출력 유형에 따라 n개의 샘플을 학습, 검증 데이터를 위해 각각 추출하여 사용하였다.

  4. KLUE-NER의 경우 ['B-PS', 'I-PS', 'B-LC', 'I-LC', 'B-OG', 'I-OG', 'B-DT', 'I-DT', 'B-TI', 'I-TI', 'B-QT', 'I-QT'] 레이블을 가지며, 각 레이블에 대해 중복없이 하나의 샘플을 추출하였다.

  5. 결과적으로 학습, 검증 데이터는 13개의 샘플을 갖는다.

  6. Pre-tuning된 PLM을 generator로 사용한 M3 실험을 KLUE-NER에 대해 수행하였을 때, 여전히 입력에 나타난 태그를 그대로 생성한다던지 태그를 덜 생성하는 문제가 있지만, 이전보다는 주어진 입력에 대해 태그를 생성하는 샘플 수가 증가하였다.

  7. NER의 경우 개체 추출과 레이블 태깅을 분리하여 수정된 실험을 진행하여 성능 향상이 가능한지 확인할 예정이다.

[MDPI2023-03] 배치 샘플링 스케줄러

  1. 2021년 HCLT에 제출한 "불균형 범주 분류를 위한 동적 샘플링 스케줄러" 논문을 확장하여 MDPI에 제출한다.

    1. HCLT2021 논문 파일: conference#42

01. 서론 및 관련연구

  1. 카테고리 불균형이 존재하는 데이터에 효과적으로 동작하는 샘플링 방법을 제안하는 것이 목적이다.

  2. 기존의 배치 샘플링 방법이 불균형 문제를 해결하는데 효과적이지 않다는 것을 언급한다.

02. 제안 방법 및 실험

  1. 저빈도 카테고리 샘플의 추출 확률을 높이는 것을 목표로 한다.

  2. 전체 데이터에서 각 카테고리의 샘플 수를 기반으로 추출 확률을 부여하는 방법(LWRS, Label Weight Random Sampling)을 제안한다.

    1. 정의된 추출 확률은 샘플별로 부여된다.

    2. 같은 카테고리인 샘플들은 항상 동일한 추출 확률을 갖는다.

  3. Reciprocal이 Ratio보다 저빈도와 고빈도 레이블의 확률 값에서 큰 차이를 보인다.

  1. 샘플에 대한 가중치가 고정되어있을 경우 과적합이 발생할 수 있어, learning rate 스케줄러를 차용하여 특정 주기마다 전체 샘플에 대한 학습을 진행하도록 한다.

03. 결론

  1. 카테고리 불균형 데이터의 분류 태스크에서 발생하는 편향 문제를 해결하고자 카테고리 개수 기반 샘플링 확률 계산 방법을 제안한다.

  2. 저빈도 카테고리의 샘플링 확률을 높여 학습에 반영을 많이하여 F1-score을 개선하였다.

  3. 고정된 샘플 추출 확률로 인해 발생할 수 있는 또 다른 편향 문제를 완화시키기 위해 배치 샘플링 방법을 전환하는 스케줄링 방법을 제안한다.

  4. 제안된 스케줄링 방법을 적용하여 F1-Score 성능을 개선하되 Accuracy 성능이 크게 하락하지 않도록 하였다.

  5. 제안된 방법은 같은 카테고리 내 샘플은 같은 확률값으로 고정하므로 샘플 간 난이도 차이를 반영하지는 않는다.

  6. 향후에는 카테고리의 크기와 샘플 난이도를 함께 반영할 수 있도록 추출 확률 계산 방법에 대한 연구가 필요하다.
